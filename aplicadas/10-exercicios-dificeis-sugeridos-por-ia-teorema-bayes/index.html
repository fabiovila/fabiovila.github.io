1 - Diagnóstico Médico com Múltiplas Condições

Uma doença rara R afeta 0,01% da população. Existem três testes disponíveis:

  Teste A: Sensibilidade = 95%, Especificidade = 98%
  Teste B: Sensibilidade = 90%, Especificidade = 99%
  Teste C: Sensibilidade = 85%, Especificidade = 99.5%

Os testes são condicionalmente independentes dado o estado da doença. Um paciente faz os três testes e:

  Teste A: Positivo
  Teste B: Negativo  
  Teste C: Positivo

Pergunta: Qual a probabilidade de o paciente ter a doença R?

Resolução

Esse tipo de exercício é recorrente na medicina. Os termos Sensibilidade e Especificidade são indicadores de eficiência conhecidos na medicina.

Sensibilidade é a proporção de pessoas com a doença em que o teste resultou positivo. Também interessa os casos onde o paciente está doente e o teste indicou negativo quando temos um caso de falso-negativo, é desejável, então, que o teste tenha Sensibilidade de 100%, ou seja, não resulte em falsos-negativo.

Especificidade é a proporção de pessoas sem a doença que o teste resultou negativo. Novamente, o desejado é que a proporção seja alta de forma que o teste não resulte em falsos-positivo.

Incialmente temos o dado que a doença afeta 0,01% da população, ou seja, temos uma taxa de prevalência igual a 0,01%, o que nos diz a priori que o paciente tem 0,01% chances de estar doente, logo

P(D) = 0,01%

P(¬D) = 100% - 0,01% = 99,9%

Para ajudar a compreender o que se pede vamos montar uma tabela para auxiliar a entender as relações entre Teste x Indicador:

Teste Sensibilidade (Verdadeiro Positivo) Especificidade (Verdadeiro Negativo) Falso Negativo (1 - Sensibilidade) Falso Positivo (1 - Especificidade)
  P(T+∣D) P(T-∣¬D) P(T-∣D)=1-P(T+∣D) P(T+∣¬D)=1-P(T-∣¬D)
Teste A 95% 98% 5% 2%
Teste B 90% 99% 10% 1%
Teste C 85% 99,5% 15% 0,5%

Exercício 2: Paradoxo da Sala de Aula Bayesiana

Em uma universidade, há três tipos de professores:

  • Tipo A (30%): Probabilidade de dar aula interessante = 0.9
  • Tipo B (50%): Probabilidade de dar aula interessante = 0.6
  • Tipo C (20%): Probabilidade de dar aula interessante = 0.3

Você assiste a 5 aulas de um professor desconhecido e observa a seguinte sequência: Interessante, Chata, Interessante, Interessante, Chata

Assumindo independência entre as aulas, calcule:

  1. A probabilidade posterior de cada tipo de professor
  2. A probabilidade da próxima aula ser interessante
  3. Quantas aulas consecutivas interessantes seriam necessárias para ter 90% de certeza de que é um professor Tipo A?

Exercício 3: Rede Bayesiana Complexa

Considere a seguinte rede causal:

  • Chuva (C): P(C) = 0.3
  • Aspersor (A): P(A) = 0.4
  • Grama Molhada (G): P(G C,A) = 0.99, P(G C,¬A) = 0.8, P(G ¬C,A) = 0.9, P(G ¬C,¬A) = 0.01
  • Vizinho Liga (V): P(V G) = 0.7, P(V ¬G) = 0.1
  • Escorregão (E): P(E G) = 0.6, P(E ¬G) = 0.05

Você observa que:

  • O vizinho ligou (V = verdadeiro)
  • Houve um escorregão (E = verdadeiro)
  • Não está chovendo (C = falso)
Calcule: P(A V,E,¬C) - A probabilidade do aspersor estar ligado.

Exercício 4: Problema da Moeda Viciada com Prior Conjugado

Você tem uma moeda com probabilidade θ de dar cara. Seu prior sobre θ é Beta(2,3).

Você joga a moeda 20 vezes e obtém a sequência: CKCCKCCKCKKCKKKCKCKC (onde C = cara, K = coroa)

  1. Calcule a distribuição posterior de θ
  2. Qual a probabilidade de θ > 0.6?
  3. Se você jogar a moeda mais 10 vezes, qual a probabilidade de obter exatamente 7 caras?
  4. Quantas caras consecutivas você precisaria observar para ter 95% de certeza de que θ > 0.7?

Exercício 5: Classificação Bayesiana com Ruído

Um sistema de classificação de e-mails tem três categorias:

  • Spam (S): 40% dos e-mails
  • Promoção (P): 35% dos e-mails
  • Normal (N): 25% dos e-mails

O sistema detecta três features:

  • Palavra “grátis”: P(F1 S)=0.8, P(F1 P)=0.6, P(F1 N)=0.1
  • Múltiplos “!”: P(F2 S)=0.9, P(F2 P)=0.7, P(F2 N)=0.05
  • Link suspeito: P(F3 S)=0.85, P(F3 P)=0.2, P(F3 N)=0.02

Porém, o sistema de detecção tem ruído:

  • P(detectar feature feature presente) = 0.95
  • P(detectar feature feature ausente) = 0.08

Um e-mail é analisado e o sistema detecta F1 e F3, mas não F2.

Calcule: As probabilidades posteriores de cada categoria, considerando o ruído na detecção.

Exercício 6: Problema de Monty Hall Generalizado

Há n portas, das quais k contêm prêmios. Você escolhe m portas inicialmente. O apresentador, que sabe onde estão os prêmios, abre p portas vazias dentre as (n-m) que você não escolheu.

Dados específicos:

  • n = 10 portas
  • k = 3 prêmios
  • m = 2 portas escolhidas inicialmente
  • p = 5 portas abertas pelo apresentador (todas vazias)
  1. Qual a probabilidade de suas portas iniciais conterem exatamente 0, 1, 2 prêmios?
  2. Se você trocar suas duas portas por duas das três restantes não abertas, qual a probabilidade de ganhar mais prêmios?
  3. Qual estratégia maximiza o número esperado de prêmios?

Exercício 7: Inferência sobre Parâmetros de Distribuição

Uma fábrica produz componentes cuja vida útil segue uma distribuição exponencial com parâmetro λ. O prior sobre λ é Gamma(α=2, β=1000).

Você testa 15 componentes e observa os seguintes tempos de falha (em horas): [850, 1200, 750, 1450, 900, 1100, 650, 1350, 800, 1250, 700, 1000, 1150, 950, 1300]

  1. Encontre a distribuição posterior de λ
  2. Calcule um intervalo de credibilidade de 95% para λ
  3. Qual a probabilidade de um novo componente durar mais de 1000 horas?
  4. Quantos componentes adicionais você precisaria testar para reduzir a incerteza sobre λ pela metade?

Exercício 8: Detecção de Mudança com Modelos Hierárquicos

Um processo industrial tem taxa de defeitos que pode mudar ao longo do tempo. Modelamos isso como:

  • Estados: θt ∈ {0.01, 0.05, 0.10} (taxa de defeitos no tempo t)
  • Transições: P(θt|θt-1) dado pela matriz: 
         0.01  0.05  0.10
0.01 [0.9   0.08  0.02]
0.05 [0.1   0.85  0.05]
0.10 [0.05  0.15  0.80]
  • Prior inicial: P(θ1) = [0.7, 0.2, 0.1]

Observações de defeitos em lotes de 100 peças nos últimos 10 períodos: [1, 3, 2, 1, 8, 12, 15, 11, 9, 6]

Calcule:

  1. A sequência mais provável de estados (algoritmo de Viterbi)
  2. A probabilidade de mudança de estado em cada período
  3. A previsão para o próximo período

Exercício 9: Calibração de Especialistas

Três especialistas fazem previsões sobre eventos binários. Seus históricos mostram:

  • Especialista A: 200 previsões, 75% de acerto, tendência otimista (overconfidence = 1.2)
  • Especialista B: 150 previsões, 80% de acerto, tendência pessimista (underconfidence = 0.8)
  • Especialista C: 300 previsões, 70% de acerto, bem calibrado

Para um novo evento, eles dão as seguintes probabilidades:

  • A: 0.75
  • B: 0.40
  • C: 0.60

Usando um modelo Bayesiano que considera:

  • Histórico de cada especialista
  • Tendências de calibração
  • Correlação entre especialistas (ρAB = 0.3, ρAC = 0.1, ρBC = 0.2)

Calcule: A probabilidade consensual calibrada para o evento.

Exercício 10: Problema de Parada Ótima Bayesiana

Você está procurando apartamento e tem n = 20 apartamentos para visitar. Para cada apartamento i, você observa um valor Vi (de 0 a 100). Você deve decidir imediatamente após cada visita se aceita ou rejeita o apartamento.

Modelo:

  • Os valores seguem uma distribuição Beta(α, β)
  • Prior: α ~ Gamma(2,1), β ~ Gamma(2,1)
  • Você quer maximizar E[V] do apartamento aceito

Você visitou 8 apartamentos com valores: [65, 78, 45, 89, 52, 71, 83, 60]

Determine:

  1. A distribuição posterior dos parâmetros α e β
  2. A regra de parada ótima para os próximos apartamentos
  3. A probabilidade de encontrar um apartamento com valor > 85 nos próximos 5
  4. O valor esperado do melhor apartamento que você conseguirá

Soluções Detalhadas

Solução do Exercício 1:

Aplicando Bayes com múltiplas evidências:

P(R) = 0.0001 P(A+|R) = 0.95, P(A-|R) = 0.05 P(B+|R) = 0.90, P(B-|R) = 0.10
P(C+|R) = 0.85, P(C-|R) = 0.15

P(A+ ¬R) = 0.02, P(A- ¬R) = 0.98
P(B+ ¬R) = 0.01, P(B- ¬R) = 0.99
P(C+ ¬R) = 0.005, P(C- ¬R) = 0.995

Likelihood com R: L(R) = 0.95 × 0.10 × 0.85 = 0.08075 Likelihood sem R: L(¬R) = 0.02 × 0.99 × 0.005 = 0.0000999

P(R|A+,B-,C+) = (0.0001 × 0.08075) / (0.0001 × 0.08075 + 0.9999 × 0.0000999) = 0.000008075 / (0.000008075 + 0.0000999) = 0.000008075 / 0.000108 = 0.0748 ou 7.48%

Solução do Exercício 2:

Sequência observada: I-C-I-I-C

Para cada tipo, calculamos a likelihood:

  • Tipo A: L(A) = 0.9 × 0.1 × 0.9 × 0.9 × 0.1 = 0.00729
  • Tipo B: L(B) = 0.6 × 0.4 × 0.6 × 0.6 × 0.4 = 0.03456
  • Tipo C: L(C) = 0.3 × 0.7 × 0.3 × 0.3 × 0.7 = 0.01323

Evidência total: E = 0.3×0.00729 + 0.5×0.03456 + 0.2×0.01323 = 0.0219687

Posteriores:

  • P(A dados) = (0.3 × 0.00729) / 0.0219687 = 0.0995
  • P(B dados) = (0.5 × 0.03456) / 0.0219687 = 0.7866
  • P(C dados) = (0.2 × 0.01323) / 0.0219687 = 0.1204

P(próxima interessante) = 0.0995×0.9 + 0.7866×0.6 + 0.1204×0.3 = 0.5972
